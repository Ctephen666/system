# 香薰系统语音集成

## 目标

将 `Ctephen666/aroma_system` 的配方生成、分阶段香薰执行和继电器控制能力接入 py-xiaozhi 的 MCP 工具体系。用户可在普通语音会话中进入香薰对话、描述需求并启动香薰；退出时停止输出并回到正常聊天。

## 范围

- 新增 `src/mcp/tools/aroma/`，通过 `@mcp_tool` 自动注册香薰工具。
- 提供进入、按自然语言需求启动、查询状态、停止/退出四类工具；工具说明中明确中文触发语和多轮会话规则。
- 复用上游项目的 16 路 DAM1600C 串口继电器协议与“阶段结束即关闭通道”的安全语义；阻塞串口 I/O 必须移出事件循环。
- 复用上游项目的本地规则配方作为无云端密钥时的可用兜底；不复制其 PyQt/QML、数据库、远程 API 或任何密钥。
- 使用小智的 `ConfigManager` 配置串口、通道映射和硬件开关。硬件控制默认关闭，未配置或依赖缺失时返回可读错误且不得写入串口。
- 用模块级会话状态表示当前是否处于香薰对话。进入后工具结果要求助手询问用户的情绪/场景；退出时中止正在执行的任务并清除模式。

## 语音体验

1. 用户说“开启香薰系统/进入香薰模式”时，模型调用进入工具，回复“香薰系统已开启，请告诉我想放松、专注、提神或助眠等需求”。
2. 用户描述需求后，模型调用启动工具；工具生成并执行配方，返回已启动的通道、总时长和配方摘要。
3. 用户说“停止香薰/退出香薰系统/关闭香薰模式”时，模型调用退出工具，所有已打开通道关闭，模型回复已退出并恢复正常聊天。

## 验收标准

- MCP 自动发现后可列出 `aroma.enter`、`aroma.start`、`aroma.status`、`aroma.exit`（或同等职责的唯一名称）。
- `aroma.enter` 和 `aroma.exit` 的返回值能表达模式状态与后续对话指令。
- `aroma.start` 只在香薰模式内执行；空需求、未启用硬件、串口失败均不抛出到会话层，返回明确 JSON 结果。
- 任务完成、失败或退出后，所有本次启动的通道均被关闭；退出不得遗留后台线程。
- 不依赖 PyQt；新增逻辑能通过 ruff 检查，并以不访问真实串口的测试覆盖会话、配方和关断行为。

## 假设与待确认

- 小智客户端与继电器控制板在同一台主机上运行，使用 DAM1600C 兼容串口协议。
- 初始版本只接入语音/MCP，不迁移香薰系统的 GUI、数据库、手机远程控制与云端模型密钥。

## 扩展范围：项目结构与配置整理

在不移除 GUI、CLI、GPIO、协议、模型、资源或已注册 MCP 功能的前提下，整理 py-xiaozhi 的项目结构。

- 删除经审计确认可再生的 Python 缓存和构建缓存，并补全忽略规则以避免重新纳入工作区。
- 将默认配置定义从 `ConfigManager` 的加载逻辑中分离，保留现有用户配置路径和所有配置键的兼容性。
- 提供不含任何密钥或设备标识的统一配置模板，覆盖系统、音频、唤醒词、日志、相机、快捷键和香薰。
- 为配置目录、配置模板和安全清理方式补充简洁文档。
- 不删除无法证明无用的业务文件、模型、资源、跨平台脚本或打包文件。

### 扩展验收标准

- 项目运行时仍只读取一个用户配置文件，且旧配置可与新默认值递归合并。
- 仓库内存在可复制的统一配置模板，模板不包含任何真实密钥、令牌、设备 ID 或个人路径。
- 已删除的内容仅限可再生缓存/构建产物或有明确无引用证据的文件。
- `python -m compileall`、配置加载与香薰 MCP 自动发现通过。

### 部署配置位置

为支持鲁班猫-4 的仓库内直接部署，运行时优先使用仓库根目录的
`config/config.json` 作为唯一显式配置文件。该文件使用 UTF-8 JSON，包含完整默认配置
且不含真实密钥。只有当仓库配置不存在或不可访问时，才回退到原有用户数据目录配置；
保留环境变量显式路径作为运维覆盖入口。README 必须写明该优先级和密钥不能提交的风险。
